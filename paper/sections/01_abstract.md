This paper studies hot/cold data classification in in-memory databases, where storing cold tuples in expensive memory can degrade resource efficiency. Prior hybrid filtering approaches combine tuple-level and column-level access signals using fixed heuristic rules. We propose an ML-guided variant that uses lightweight classifiers to predict tuple hotness from access pattern features. A reproducible synthetic workload benchmark is developed in Python with in-memory execution and no cluster dependency. We compare a decision-tree or logistic-regression classifier against an HFA-like heuristic baseline. Results are reported in terms of classification quality and downstream indicators relevant to memory tiering. The study highlights how low-overhead ML can improve hot/cold prediction in practical undergraduate-scale experiments.
